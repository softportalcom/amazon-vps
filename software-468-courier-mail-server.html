СКАЧАТЬ

Courier Mail Server — это почтовый сервер под Windows для офисных и домашних сетей. Компактный, простой в настройке и эксплуатации, многопоточный, содержит Web- и SOCKS-сервера. Он поможет вам быстро организовать обмен электронной почтой в локальной сети и в Интернете.

Что нового в Courier Mail Server 3.10?
  • Журнал работы сервера разделён на 6 журналов: основной и для каждого сервиса. Каждый журнал сохраняется в отдельный файл и отображается на отдельной вкладке в консоли. Вкладки неиспользуемых (пустых) журналов не отображаются.
  • Автоматическое удаление старых файлов журнала (хранящихся больше заданного числа дней).
  • Возможность выбирать вид и размер шрифтов в параметрах консоли (основной шрифт, шрифт журнала и текста шаблона письма, шрифт заголовков разделов).
  • Для каждого внешнего IMAP-ящика теперь можно указать папку, из которой будет приниматься почта.
  • Возможность ограничить для каждого внешнего SMTP-сервера максимальное число писем, отправляемых на него за указанное время.
  • Возможность ограничить для каждого внешнего SMTP-сервера максимальное число получателей в каждом отправляемом на него письме.
  • Возможность запретить проверку сертификата сервера при установке защищённых соединений (SSL/TLS) с внешними серверами, в том числе при MX-доставке.
  • Если серверу не удаётся открыть файл журнала, то в консоли вкладка этого журнала помечается и в ней выводится информация об ошибке. Когда файл успешно открыт, во вкладке выводится имя текущего файла журнала.
  • Возможность выбирать собственный уровень подробности журнала для каждого сервиса.
  • Поле "Папка файлов журнала" в разделе "Настройки - Журнал".
  • Теперь максимальный размер файла журнала всегда ограничен и может задаваться как в килобайтах, так и в мегабайтах.
  • Теперь сохраняются все файлы журнала, достигшие максимального размера, а не только последний (*.old), как было ранее.
  • Удалена настройка журнала "Режим сохранения в файл". Теперь всегда действует режим "Отдельный файл для каждой даты".
  • Удалён флажок "Записывать протокол администрирования" в разделе "Настройки - Журнал". Теперь этот протокол записывается в журнал WinAdmin-сервиса при высоком и отладочном уровнях подробности.
  • Небольшие улучшения, исправление мелких ошибок и недочётов в консоли, сервере и установщике (конвертер конфигурации).
  • Некоторые принятые письма IMAP-клиент сохранял с нарушением формата, что могло привести в дальнейшем к передаче этих писем в неполном виде.
  • Ошибки координации потоков при отправке почты.
