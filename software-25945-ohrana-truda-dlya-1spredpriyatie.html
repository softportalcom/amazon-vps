"Охрана труда" для 1С:Предприятие - прикладное решение на платформе 1С:Предприятие 8.3 для автоматизации рабочего места инженера по охране труда. Данный продукт предназначен для автоматизации проведения аттестации рабочих мест и медицинских осмотров с учетом последних изменений законодательства.

Особенности:

  • Учет ведется в соответствии с "новыми" приказами.
  • Повышается эффективность труда инженеров по охране труда, в их деятельности появляется систематичность.
  • Накапливается «небухгалтерская» статистическая информация для принятия управленческих решений.
  • Снижается риск штрафных санкций со стороны контролирующих органов (Роспотребнадзор, Санэпидемстанция, Инспекция труда).

Функциональные возможности:

  • Подсистема "Медицинские осмотры":
    • формирование перечня контингентов,
    • формирование списка лиц для медосмотров,
    • формирование графика медосмотров,
    • расчет стоимости медосмотров,
    • ведение паспортов здоровья.
  • Подсистема "Аттестация рабочих мест" по условиям труда.
    • Формирование рабочих мест. Возможность автоматического формирования кода рабочего места. Возможность автоматического формирования рабочих мест по штатному расписанию или по основному оборудованию.
    • Формирование комиссии для проведения аттестации рабочих мест.
    • Формирование графика замеров.
    • Отражение замеров на рабочих местах. Замеров вредных факторов, показателей тяжести и напряженности трудового процесса, требований безопасности.
    • Производственный контроль.
    • Оценка обеспеченности средств индивидуальной защиты (СИЗ) на рабочих местах.
    • Комплексная оценка состояния условий труда на рабочем месте.
    • Формирование карты аттестации рабочего места.
    • Формирование необходимых печатных документов по аттестации рабочих мест.
  • Возможность вести учет по нескольким организациям.
  • Возможность загрузки данных через COM-соединение из конфигураций 1С:Зарплата и управление персоналом 8, 1С:Комплексная автоматизация и 1С:Управление производственным предприятием 8.
  • Возможность загрузки данных из XML и табличных документов.
  • Возможность загрузки данных в специальном формате, предназначенном для организации регулярных обменов данными с любыми программами.
  • Хранение нормативной документации, как в базе, так и в указанном каталоге.
  • Механизм внешних отчетов, обработок, печатных форм и обработок заполнения табличных частей документов.
  • Работа в многопользовательском режиме.
  • Рабочая база данных поставляется с заполненной нормативно-справочной информацией (НСИ). Это позволяет сразу приступить к ведению учета и значительно снизить затраты на ввод программы в эксплуатацию.
